# GTD

## Resumen

Metodo de productividad para sacar compromisos de la cabeza, procesarlos en un sistema confiable y ejecutar acciones segun contexto, tiempo y prioridad real.

## Ideas clave

- Toda captura debe pasar por una o varias bandejas de entrada.
- Cada item se procesa para decidir si requiere accion.
- Lo no accionable se desecha, se incuba o se guarda como referencia.
- Lo accionable se hace, se delega, se agenda o se convierte en siguiente accion.
- Si requiere varias acciones, debe tratarse como proyecto.
- El sistema solo funciona si se revisa con regularidad.

## Fragmentos migrados

- Si una accion toma menos de 2 minutos, se hace al momento.
- Las acciones delegadas deben ir a seguimiento.
- Las acciones futuras van al calendario o a alguna lista contextual.
- La revision semanal sostiene la confianza en el sistema.
